⚠️ Error de CORS - Solución

Problema: Estás abriendo el archivo HTML directamente desde el explorador de archivos (file://), lo cual causa errores de CORS con Firebase Storage.

Soluciones:

Opción 1: Usar Python (Recomendado - Más fácil)

1. Abre la terminal/cmd en la carpeta del proyecto

2. Ejecuta:

python -m http.server 8000

3. Abre tu navegador en: http://localhost:8000/agregar-producto.html

Opción 2: Usar Node.js

1. Abre la terminal/cmd en la carpeta del proyecto

2. Ejecuta:

npx http-server -p 8000

3. Abre tu navegador en: http://localhost:8000/agregar-producto.html

Opción 3: Usar Visual Studio Code

1. Instala la extensión "Live Server" en VS Code

2. Haz clic derecho en agregar-producto.html

3. Selecciona "Open with Live Server"

Opción 4: Usar PHP

1. Abre la terminal/cmd en la carpeta del proyecto

2. Ejecuta:

php -S localhost:8000

3. Abre tu navegador en: http://localhost:8000/agregar-producto.html

¿Por qué es necesario?

Firebase Storage requiere que las peticiones vengan de un origen HTTP válido (http:// o https://), no desde archivos locales (file://). Por eso necesitas un servidor web local.

Después de iniciar el servidor:

Una vez que tengas el servidor corriendo, podrás subir imágenes sin problemas. El error de CORS desaparecerá porque ahora el origen será http://localhost:8000 en lugar de null.
